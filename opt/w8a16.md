# W8A16 主线

## 定位

W8A16 是简单的省显存 decode 模式。

- 权重：FP8。
- 激活：BF16。
- Runtime：只保留一条基础 Triton on-the-fly dequant GEMM 路径。
- 核心 claim：checkpoint / 权重占用更小，同时有一个面向 decode 的 runtime 故事。

这条线要保持简单。不要把 W8A16 重新做成很多 backend 和特例的集合。

## 保留

### Triton 基础路径

W8A16 主线只保留这一条实现：

```text
FP8 weight + BF16 activation -> Triton on-the-fly dequant GEMM
```

这已经足够支撑 W8A16 的故事：

- FP8 权重降低模型占用。
- BF16 激活保留稳定的质量路径。
- Triton 提供直接的 decode-oriented runtime，不隐藏 BF16 weight cache 之类的技巧。

## 不进主线

下面这些不再作为 W8A16 的 forward 主线：

- per-forward BF16 dequant matmul 作为推广路线。
- BF16 weight cache。
- hybrid cache 变体。
- W8A16 CUDA/PTX 原型。
- down-projection 或 layer-name 特例。
- large-prefill 优化 claim。

如果以后需要追溯，它们可以存在于历史记录或提交历史里，但不要出现在 active choice、README claim 或默认 runtime policy 里。

## 验证口径

W8A16 用一组很直接的 gate 评估：

| Gate | 目的 |
| --- | --- |
| Checkpoint size | 证明 FP8 权重的显存/体积收益 |
| PPL / logits | 证明质量接近 BF16 |
| Decode latency | 证明 Triton 路径对 decode 有价值 |
| Prefill latency | 如实报告，但不要围绕 prefill 优化这条线 |

## 7B 远端验证

远端环境：1x RTX 4090，`/root/miniconda3/bin/python`，模型统一使用 7B。

路径：

- BF16：`/root/autodl-tmp/models/qwen/Qwen2___5-7B-Instruct`
- W8A16：`/root/autodl-tmp/models/qwen/Qwen2___5-7B-Instruct-FP8-W8A16`

W8A16 checkpoint 由本仓库 `scripts/quantization/quantize.py --scheme fp8_w8a16` 从 BF16 7B 导出，量化了 `252/434` 个 tensor。

| Case | Model size bytes | After-load allocated | After-load reserved | Workload | Decode TPS |
| --- | ---: | ---: | ---: | --- | ---: |
| BF16 memory | `6,183,464,346` | `41,249,649,664` | `41,393,586,176` | input 512, output 16 | `130.69` |
| W8A16 memory | `3,413,055,132` | `41,027,154,944` | `41,190,162,432` | input 512, output 16 | `158.36` |
| W8A16 decode | `3,413,055,132` | `41,027,154,944` | `41,190,162,432` | input 512, output 128, warmup 1, repeat 3 | `158.73` |

结论：

- W8A16 checkpoint/model tensor size 从 BF16 的 `6.18 GB` 降到 `3.41 GB`，约 `44.8%` 更小。
- 当前 512 prompt / decode 场景下，W8A16 Triton 路径 decode 从 BF16 `130.69 tok/s` 提升到 `158.36 tok/s`，约 `1.21x`。
- 128-token decode repeat 的 W8A16 decode 为 `158.73 tok/s`。
- CUDA allocated/reserved 的端到端差异小于 checkpoint 差异，说明当前测试还受 KV/allocator/runtime 常驻开销影响；README 只应把 W8A16 讲成 weight-only 省占用和 decode 改善，不应扩大成整卡显存减半。

证据已拉回本地：

- `.remote-logs/quantization/7b_mainline/bf16_memory.json`
- `.remote-logs/quantization/7b_mainline/w8a16_memory.json`
- `.remote-logs/quantization/7b_mainline/w8a16_decode.json`

## 非目标

- 不用 W8A16 讲 W8A8 的 kernel 优化故事。
- 不给 W8A16 增加自动 backend selection。
- 不同时维护多条 W8A16 kernel 路径，除非某条只是历史代码且不接入主线。
